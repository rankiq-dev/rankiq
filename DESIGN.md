## 1. Archetype

**Cinematic Dark Pro** — reference: Linear, Vercel Dashboard, Raycast.

Deep navy canvas + teal-cyan gradient accent + glassmorphism surfaces + Three.js particle field on hero/landing.
Motion is purposeful and cinematic: GSAP entrance sequences (stagger, count-up), not decorative loops.
Data is the hero. The UI recedes; numbers and status glow.

**Why this archetype for RankIQ:**
SEO is analytical and trust-sensitive. A cinematic dark theme signals precision and authority — the same register as monitoring tools that handle serious data. The teal-cyan accent (hue 175–195) is distinct from both the red "danger" of error dashboards and the blue of generic SaaS, while still reading as "technical + growth-oriented."

## 4. Motion System

**Philosophy:** Motion signals meaning, not decoration. Every animation has a job.

| Tier | What | How | When |
|---|---|---|---|
| T0 — Micro | Hover states, focus rings | CSS transition 150ms `ease-out` | Always on |
| T1 — Response | Button press, tab switch | CSS transition 200–250ms `ease-out` | Always on |
| T2 — Entrance | Page / panel mount | GSAP `power3.out` 450–700ms, stagger 70ms | On mount |
| T3 — Cinematic | Score ring draw, count-up, particle field | GSAP timeline + Three.js | Hero sections only |

**GSAP entrance sequence (Audit Results):**
1. Sidebar slides in from left (`x: -20 → 0`, 700ms)
2. Topbar fades in (500ms, overlapping -300ms)
3. Score ring card rises (`y: 30 → 0`, 700ms)
4. Stat cards stagger up (450ms, 100ms apart)
5. Ring stroke draws (`strokeDashoffset: 408 → target`, 1400ms `power2.out`)
6. Score counter ticks up (1200ms, parallel with ring)
7. Table rows stagger in from left (`x: -16 → 0`, 70ms per row)

**Reduced motion:** All T2/T3 animations respect `prefers-reduced-motion: reduce` — skip directly to end state.

## 5. Layout System

**Shell:** Fixed sidebar (210px) + sticky topbar (52px) + scrollable content area.
**Grid:** 4px base unit. 24px page padding. 16px card gap, 12px inner gap.
**Background:** Three.js particle canvas `position:fixed` behind all content (z-index 0). UI surfaces use `backdrop-filter: blur(20px)` to sit above it.

| Breakpoint | Layout |
|---|---|
| ≥ 1024px | Sidebar visible + full topbar |
| 768–1023px | Sidebar hidden (sheet/drawer on demand) |
| < 768px | Bottom nav; table collapses to 2 columns |

## 6. Component Patterns

### Cards / Surfaces
- `background: var(--glass-bg)` + `backdrop-filter: var(--glass-blur)`
- `border: 1px solid var(--glass-border)`
- `border-radius: var(--radius-lg)` (12px default for cards)
- Top-edge accent line (`height: 2px`, gradient from `--primary` to transparent) for status cards

### Buttons
| Variant | Style |
|---|---|
| Primary | `background: linear-gradient(135deg, var(--primary), var(--primary-2))` + `--shadow-glow` on hover |
| Ghost | `background: oklch(1 0 0 / 0.04)`, `border: 1px solid var(--glass-border)` |
| Accent-soft | `background: var(--primary-soft)`, `color: var(--primary-2)` |

### Severity Badges
| Level | Color | Glow |
|---|---|---|
| Critical | `--destructive` | `drop-shadow(0 0 6px var(--destructive))` |
| Warning | `--warning` | `drop-shadow(0 0 6px var(--warning))` |
| Info | `--primary-2` | `drop-shadow(0 0 6px var(--primary-glow))` |
| Success | `--success` | `drop-shadow(0 0 6px var(--success))` |

### Score Ring
SVG `<circle>` with `strokeDashoffset` driven by GSAP. Stroke = `linear-gradient(--primary → --primary-2)` via SVG `<linearGradient>`. `filter: drop-shadow(0 0 6px var(--primary-glow))`.

## 7. Three.js + GSAP Spec

**File:** `src/components/features/audit/ScoreRingCanvas.tsx` (Three.js instance)
**CDN in sample:** `three.js r134`, `gsap 3.12.5`
**Production:** `npm install three gsap`

**Particle field:**
- 1 800 points, `THREE.PointsMaterial`, `vertexColors: true`
- Color palette: `#0d9488`, `#06b6d4`, `#14b8a6`, `#22d3ee`, `#5eead4`
- Mouse parallax: `particles.rotation.y += mouseX * 0.08`, `rotation.x += mouseY * 0.05`
- Y drift: `Math.sin(t * 0.3) * 0.1`
- Speed multiplier: 0 / 0.3 / 1.2 (Off / Slow / Fast)

**GSAP timeline entry point:** `src/components/features/audit/auditEntrance.ts`
All animated elements use `data-anim` attributes; GSAP selects by those, never by class names that might change.

## 8. Accessibility

- **WCAG AA minimum** on all text/background pairs (verified in OKLCH with contrast ≥ 4.5:1 on text, ≥ 3:1 on UI elements)
- **Focus ring:** `outline: 2px solid var(--primary); outline-offset: 2px` on all interactive elements
- **Reduced motion:** `@media (prefers-reduced-motion: reduce)` sets all GSAP durations to 0
- **Colour-blind safe:** status meaning never conveyed by hue alone — always paired with icon or text label
- **Screen reader:** Three.js canvas has `aria-hidden="true"`; score ring has `role="img" aria-label="Site health score: 68 out of 100"`
