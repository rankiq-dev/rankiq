# M3 — On-Page SEO Analysis

## What
Per-URL on-page scoring (0–100) added to every audit. The crawler now captures image alt coverage, JSON-LD schema presence, and H3 counts. After crawling, a second pass computes incoming internal link counts per page. The analyzer produces per-page `PageAnalysis` records stored as JSONB on the audit row. The audit results UI shows a sortable URL table (score + word count + heading counts + issue count) and a health score ring chart.

## On-Page Score Formula (per URL, 0–100)

| Deficiency | Penalty |
|---|---|
| Missing title | -20 |
| Missing H1 | -15 |
| Missing meta description | -10 |
| Noindex (unexpected) | -30 |
| Title too long/short | -5 each |
| Multiple H1s | -5 |
| No canonical | -5 |
| No heading hierarchy (no H2, ≥300 words) | -5 |
| Thin content (<300 words) | -5 |
| Images missing alt | -3 |
| Poor internal linking | -5 |
| Meta description too long | -3 |

## New API Endpoint

`GET /api/v1/audits/:id/pages?sortBy=onPageScore&order=asc`

Returns: `{ data: { pages: PageAnalysis[], total: number } }`

Returns 409 if audit is not yet `complete`.
